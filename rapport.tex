\chapter{Introduction}

Ce projet s'inscrit dans le cadre du module d'intelligence artificielle en Master 1. L'objectif est de classer des séries temporelles issues de la supervision de systèmes informatiques. Ces données représentent l'activité de composants essentiels comme le processeur (CPU), le disque et la mémoire vive (RAM).

L'enjeu est de pouvoir identifier automatiquement le type d'activité à partir de traces d'exécution. Dans un contexte réel, cela permettrait de surveiller l'état de santé d'un serveur ou d'identifier des comportements anormaux.

\section{Objectifs du projet}
Le travail demandé consiste à tester et comparer trois approches différentes :
\begin{itemize}
    \item Les réseaux de neurones récurrents de type LSTM.
    \item Les réseaux de neurones convolutionnels (CNN 1D).
    \item Les modèles plus récents basés sur le mécanisme d'attention (Transformers).
\end{itemize}

\chapter{Présentation et préparation des données}

\section{Origine des données}
Les données utilisées proviennent d'un dépôt public regroupant des traces d'exécutions d'outils de tests de charge. Chaque échantillon est un fichier CSV contenant plusieurs colonnes de mesures système prises à intervalles réguliers.

\section{Exploration visuelle}
Avant de traiter les données, nous avons visualisé les signatures caractéristiques de chaque catégorie. Par exemple, une activité CPU intense se traduit par une charge élevée et constante sur le processeur, tandis qu'une activité disque montre des pics lors des phases de lecture ou d'écriture.

\section{Prétraitement}
Pour que les modèles puissent apprendre efficacement, les données subissent plusieurs transformations :
\begin{itemize}
    \item \textbf{Sélection des colonnes} : Nous retenons l'usage global du CPU, l'occupation de la RAM et les taux de lecture/écriture disque.
    \item \textbf{Normalisation} : Les valeurs sont ramenées entre 0 et 1 pour éviter que les grandes unités (comme les octets de mémoire) n'écrasent les petites (pourcentages CPU).
    \item \textbf{Fenêtrage} : Les données sont découpées en séquences de 50 unités de temps. C'est ce bloc de 50 points qui sera donné en entrée aux modèles.
\end{itemize}

\chapter{Modèle 1 : LSTM (Long Short-Term Memory)}

\section{Théorie et fonctionnement}
Le LSTM est un type de réseau de neurones récurrent (RNN) conçu pour traiter des données séquentielles. Contrairement à un réseau classique, il possède une "mémoire" lui permettant de retenir des informations sur le long terme. Cette architecture est idéale pour nos traces système, car un pic d'activité CPU n'a de sens que s'il est analysé dans son contexte temporel.

Le fonctionnement repose sur des "portes" (gates) qui décident quelles informations passées doivent être conservées, mises à jour ou oubliées.

\section{Architecture implémentée}
Pour notre projet, nous avons construit un modèle avec les caractéristiques suivantes :
\begin{itemize}
    \item \textbf{Couche LSTM} : 64 unités pour capturer la dynamique des 4 variables d'entrée.
    \item \textbf{Dropout (20\%)} : Une sécurité pour éviter le sur-apprentissage en désactivant aléatoirement certains neurones pendant l'entraînement.
    \item \textbf{Couche Dense} : 32 neurones avec activation ReLU pour la prise de décision.
    \item \textbf{Couche de sortie} : 3 neurones avec activation Softmax pour obtenir une probabilité sur les classes (CPU, Disque, Mémoire).
\end{itemize}

L'entraînement a été réalisé sur 15 époques avec une taille de lot (batch size) de 64.

\section{Résultats et analyse}
Le modèle obtient une précision globale de \textbf{91\%}. Cependant, l'analyse détaillée montre des disparités selon les classes.

\subsection{Performance par catégorie}
\begin{itemize}
    \item \textbf{Disque} : Le score est parfait (F1-score de 1.00). La signature des accès disque est très marquée et ne laisse aucune place à l'erreur.
    \item \textbf{CPU} : Très bien identifié avec un score de 0.92.
    \item \textbf{Mémoire} : C'est le point faible du modèle avec un rappel (recall) de seulement \textbf{0.59}. Une partie des activités mémoire est confondue avec le CPU. Cela s'explique par le fait que certaines opérations de gestion de la RAM sollicitent aussi le processeur d'une manière similaire à un calcul de fond.
\end{itemize}

\chapter{Modèle 2 : CNN 1D (Convolutional Neural Network)}

\section{Théorie et fonctionnement}
Habituellement utilisé pour l'analyse d'images (2D), le réseau de neurones convolutionnel (CNN) peut être adapté aux séries temporelles en utilisant des convolutions 1D. Au lieu de regarder l'image entière, le modèle fait glisser des filtres le long de la série temporelle pour détecter des motifs locaux (pics, transitions brusques, oscillations spécifiques).

L'avantage du CNN par rapport au LSTM est sa capacité à détecter des signatures de formes précises sans avoir besoin de "retenir" toute l'histoire de la séquence.

\section{Architecture implémentée}
Le modèle CNN 1D repose sur l'architecture suivante :
\begin{itemize}
    \item \textbf{Couche Conv1D} : 64 filtres avec une taille de noyau (kernel) de 3 pour extraire les caractéristiques locales.
    \item \textbf{MaxPooling1D} : Réduit la dimension temporelle en ne gardant que les valeurs les plus significatives de chaque zone.
    \item \textbf{Flatten} : Transforme les cartes de caractéristiques en un vecteur plat pour la classification finale.
    \item \textbf{Dense et Dropout} : Couches classiques pour l'interprétation finale et la régularisation.
\end{itemize}

\section{Résultats et analyse}
Le CNN 1D montre une légère amélioration par rapport au LSTM avec une précision globale de \textbf{92\%}.
